# 살아있는 숲

현재 안정판: **V1.4 stable**  
이전 안정판: **V1.3 stable**  
최근 업데이트: 방문자 등장 조건 개선 안정화  
공개 주소: https://inhm1234.github.io/living_forest/

## 프로젝트 소개

살아있는 숲은 하루에 한 번 지금의 마음을 남기면, 내 나무와 월드 숲이 조용히 자라는 웹 기반 체험판입니다.

현재 버전은 서버 저장이나 로그인 없이, 브라우저의 `localStorage`에 기록을 저장합니다. 그래서 기록은 이 기기와 이 브라우저에만 남습니다.

## 현재 포함된 기능

- 월드 숲 첫 화면
- 개인 정원 이동
- 나무 이름 정하기
- 하루 1번 감정 기록
- 좋음 / 보통 / 피곤 감정에 따른 성장 반응
- 3일 / 7일 / 14일 / 21일 / 30일 성장 목표 안내
- 월드 숲 내 자리 단계 안내
- 새 / 다람쥐 방문자 애니메이션
- 최근 방문자 기록 3개 표시
- 방문자 등장 조건 개선
- 테스트 모드 `?test=1`

## V1.4 stable 핵심 변경점

V1.4 stable은 V1.4.1 test에서 확인된 방문자 조건 개선판을 안정판으로 고정한 버전입니다.

기능을 새로 추가하지 않고, 정상 확인된 흐름을 안정판 기준으로 정리했습니다.

- 오늘 감정 기록 전에는 방문자 자동 판정을 만들지 않음
- 오늘 감정 기록을 완료한 뒤 확률에 따라 방문자 등장 가능
- 이미 오늘 기록한 사람도, 오늘 방문자 판정이 아직 없으면 개인 정원 입장 시 하루 1회 방문자 판정
- 오늘 방문자 판정이 이미 있으면 새로고침해도 다시 뽑지 않음
- 나무 이름 없음 / 3일 미만 / 기록 전 상태에서는 안내 문구만 표시
- 기존 새/다람쥐 방문 애니메이션과 방문 기록 기능은 유지
- 기존 저장키 `livingForestV012` 유지
- 테스트 저장키 `livingForestV012_TEST` 유지

## 테스트 방법

일반 화면:

https://inhm1234.github.io/living_forest/

테스트 모드:

https://inhm1234.github.io/living_forest/?test=1

테스트 모드에서 확인할 것:

- `방문 조건 기록 전` 버튼을 누르면 방문자가 바로 등장하지 않는지
- 기록 전 안내 문구가 자연스러운지
- 감정 기록 후 방문자 판정이 작동하는지
- `방문 조건 기록 후` 버튼에서 이미 오늘 기록 완료 상태의 방문자 판정이 작동하는지
- `방문자 새`, `방문자 다람쥐`, `방문 기록 3개` 버튼이 정상 작동하는지
