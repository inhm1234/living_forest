오늘의숲 · 동물 방문 v1 / DEV 화면 2단계

반영 대상: /dev/garden-login-test/
운영 루트(/)와 Cloudflare 운영판은 이번 단계에서 수정하지 않습니다.

이 파일 묶음은 앞서 Supabase에서 SQL_GARDEN_ANIMAL_VISIT_V1.sql 실행이 성공한 뒤에만 올립니다.

바뀐 점
1. 기존 localStorage 기반의 “기기마다 오늘의 동물 1마리” 결정 코드를 제거했습니다.
2. sync_my_garden_animal_visit RPC로 계정 공통 상태를 읽습니다.
3. 조용한 정원 → 20~60초 다가오는 기척(🍃) → 동물 방문 순서로 표시합니다.
4. 실제 동물 체류 시간은 서버 SQL 값(새 4분 / 토끼 8분 / 다람쥐 10분 / 고슴도치 15분)을 따릅니다.
5. 편지 전송 성공 뒤 depart_my_garden_animal_visit RPC를 호출해 모든 기기에서 같은 동물이 떠나게 합니다.
6. 창이 열려 있을 때는 도착·퇴장 시각에 한 번 즉시 갱신하고, 공통 갱신은 30초 단위입니다. 매초 서버 호출은 하지 않습니다.

검수 순서
1. 같은 계정으로 DEV에 로그인합니다. 처음에는 동물이 아니라 🍃 기척이 보이는지 확인합니다.
2. 최대 60초 안에 동물이 실제로 나타나는지 확인합니다.
3. PC와 모바일을 각각 새로 열어 같은 상태(기척 또는 같은 동물)를 보는지 확인합니다.
4. 동물에게 실제 친구 편지를 보내고, 다른 기기에서 같은 동물이 다시 대기하지 않는지 확인합니다.
5. 이번 단계에서는 URL의 ?animalPreview=... 검수 주소를 사용하지 않습니다.

주의
