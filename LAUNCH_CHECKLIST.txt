살아있는 숲 V1.1 안정판 확인 체크리스트

[일반 화면]
□ index.html 접속 시 월드 숲이 먼저 보인다.
□ 첫 화면에서 “하루 한 번 마음을 나무에 남기는 체험”임을 이해할 수 있다.
□ 일반 사용자에게 개발자 테스트 패널이 보이지 않는다.
□ “체험판 · 기록은 이 기기에만 저장돼요” 문구가 부담스럽지 않다.

[성장 목표 안내]
□ 첫 방문/0일차에서 오늘 기록을 유도하는 문구가 자연스럽다.
□ 3일차 목표인 월드 숲 작은 빛 예고가 안내된다.
□ 7일차 목표인 월드 숲 정식 자리잡기가 안내된다.
□ 14일차, 21일차, 30일차 목표 안내가 어색하지 않다.
□ 오늘 기록 완료 후 다음 목표까지 남은 일수가 보인다.

[첫 방문자 흐름]
□ 내 자리에서 시작하기 버튼을 누르면 개인 정원으로 간다.
□ 이름이 없을 때 감정 버튼은 잠겨 있다.
□ 이름 저장 후 감정 선택이 가능하다.
□ 감정 기록 후 “오늘은 여기까지” 느낌으로 마무리된다.
□ 전체 숲으로 돌아가면 내 자리에 오늘의 기운이 반영된다.

[테스트 모드]
□ index.html?test=1로 테스트 패널이 열린다.
□ 일반 화면 index.html에서는 테스트 패널이 보이지 않는다.
□ 첫 방문자 / 이름만 있음 / 오늘 기록 전 / 오늘 기록 완료 / 깊어진 나무 / 구버전 보정 프리셋이 작동한다.
□ 테스트 초기화가 작동한다.

[배포]
□ 기존 assets/garden, assets/world, assets/share 폴더를 삭제하지 않았다.
□ index.html / style.css / app.js / VERSION.txt / LAUNCH_CHECKLIST.txt / UPLOAD_GUIDE.txt를 덮어썼다.
□ GitHub Pages에서 새로고침 후 최신 화면이 보인다.
□ 테스트 모드 데이터 표시에서 V1.1 · schema 2가 보인다.

모두 통과하면: V1.1 안정판 확인 완료
